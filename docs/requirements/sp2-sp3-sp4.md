# SP2 / SP3 / SP4 需求概要

> 基于 SP1 spec 路线图、扩展点预留、以及需求澄清对话整理。

---

## SP2：组态画面（优先级：高）

**目标：** SVG 拖拽编辑 + 运行时渲染，用户可自由绘制工业组态图。
**预计复杂度：** 高

**需求：**
- [ ] 画布支持自由拖放图元（设备图标、管线、文字、矩形、圆形）
- [ ] 图元可绑定数据点（tag_id），运行时实时更新数值/颜色
- [ ] 图元库可扩展（SP4 外部 widget 包可注册新图元）
- [ ] 页面类型：SP1 仪表盘页 + SP2 组态页，两者在工程中并存
- [ ] 保存为工程文件 `dashboards/*.toml`

**依赖：** SP1 widget 注册表机制 + 实时数据订阅

---

## SP3：报警 / 长期归档 / 报表（优先级：中）

**目标：** 阈值报警、长期归档压缩、报表导出、趋势增强。
**预计复杂度：** 高

**需求：**
- [ ] 阈值报警：每个 tag 可配置报警条件（>=, <=, ==），触发时写入 alarms 表 + 前端弹窗
- [ ] 长期归档：采样压缩（变化存储而非全量 1s 入库），归档策略可配
- [ ] 报表导出：历史数据导出 CSV/Excel
- [ ] 趋势增强：多 tag 叠加、时间缩放、游标读数
- [ ] SQLite alarms 表（SP1 已预建），报警 Hook 点位（SP1 已预留）

**依赖：** SP1 Hook trait + SQLite + event-bus

---

## SP4：动态扩展 / 脚本引擎（优先级：低）

**目标：** 协议动态加载 + 脚本引擎（Lua/JS）+ 外部 widget 包加载。
**预计复杂度：** 高

**需求：**
- [ ] 协议动态加载：第三方 dylib 实现 `Source` trait，按 manifest 注册，不改 hc-core
- [ ] 脚本引擎：嵌入 Lua（rlua/mlua）或 JS（boa），在 `Hook::on_user_action` 等点位执行用户脚本
- [ ] 外部 widget 包：约定目录结构 + manifest，动态 import 渲染
- [ ] 所有扩展机制不破坏 SP1 单二进制默认形态

**依赖：** SP1 Source trait + WidgetRegistry + Hook trait + protocol_params JSON 透传

---
